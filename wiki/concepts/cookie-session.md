# 쿠키와 세션

쿠키와 세션을 사용하는 이유는 HTTP 프로토콜의 특징이자 약점(무상태성)을 보완하기 위해서다.

## 쿠키 (Cookie)

**정의:** 클라이언트(브라우저) 로컬에 저장되는 작은 데이터 파일. 키와 값으로 구성된 정보 저장소.

**주요 특징:**
- 이름, 값, 만료일(저장기간), 경로 정보로 구성되어 있다.
- 클라이언트에 총 300개의 쿠키를 저장할 수 있다.
- 하나의 도메인 당 20개의 쿠키를 가질 수 있다.
- 하나의 쿠키는 4KB(4096byte)까지 저장 가능하다.

**동작 방식:**
1. 클라이언트 요청: 사용자가 웹 페이지를 요청합니다.
2. 서버 응답: 서버는 Set-Cookie HTTP 헤더를 사용하여 쿠키 정보를 포함합니다.
3. 브라우저 저장: 브라우저는 받은 쿠키를 로컬에 저장합니다.
4. 자동 전송: 이후 같은 도메인에 대한 요청을 할 때, 브라우저는 저장된 쿠키를 자동으로 HTTP 요청 헤더에 포함시켜 서버에 전송합니다.
5. 서버 처리: 서버는 받은 쿠키 정보를 통해 사용자의 상태를 식별하고 필요한 정보를 유지하거나 업데이트합니다.

**사용 예:** 자동로그인, "오늘 더 이상 이 창을 보지 않음" 체크, 쇼핑몰의 장바구니

## 세션 (Session)

**정의:** 서버에서 클라이언트의 상태 정보를 저장하는 메커니즘. 서버 측에서 사용자를 구분하고 인증 상태를 유지하는 방법.

**주요 특징:**
- 서버 메모리에 사용자 정보를 저장
- 각 클라이언트마다 고유한 세션 ID 부여
- 브라우저가 종료되면 세션 정보 삭제
- 서버 자원을 사용하므로 대량 접속 시 성능에 영향 줄 수 있음
- 보안성이 쿠키에 비해 높음

**동작 방식:**
1. 클라이언트 접속: 사용자가 웹 서버에 접속
2. 세션 ID 생성: 서버는 해당 클라이언트에 대한 고유 세션 ID 생성
3. 세션 저장: 서버 메모리에 세션 ID와 관련 사용자 정보 저장
4. 쿠키로 전달: 생성된 세션 ID를 쿠키 형태로 클라이언트에 전송
5. 요청 시 인증: 이후 클라이언트 요청 시 세션 ID로 사용자 식별 및 인증

**사용 예:** 로그인 같이 보안상 중요한 작업을 수행할 때 사용

## 쿠키와 세션의 차이점

| 구분 | 쿠키 | 세션 |
|---|---|---|
| 저장 위치 | 클라이언트 로컬(브라우저) | 서버 메모리 |
| 보안성 | 비교적 낮음 (클라이언트에서 변조 가능) | 상대적으로 높음 (서버에서 관리) |
| 저장 용량 | 각 쿠키 최대 4KB, 총 300개, 도메인당 20개 | 서버 메모리의 용량에 의존 |
| 만료 시간 | 설정한 유효 기간까지 저장 | 브라우저 종료 시 또는 설정한 시간 후 만료 |
| 처리 속도 | 빠름 (클라이언트에서 직접 처리) | 상대적으로 느림 (서버의 데이터 조회 필요) |
| 정보 유출 위험 | 클라이언트 측에 저장되어 유출 가능 | 서버 측에 저장되어 유출 위험 적음 |
