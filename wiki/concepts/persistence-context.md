# 영속성 컨텍스트란 무엇인가요?

ORM(Object-Relational Mapping)에서 데이터베이스와 객체 간의 매핑을 관리하는 핵심 개념.

**정의:** 영속성 컨텍스트는 ORM이 객체의 상태를 추적하고 유지하기 위해 사용하는 메모리의 영역입니다. 데이터베이스 레코드와 애플리케이션 객체 간의 일관성을 보장하는 역할을 합니다.

## 주요 개념

- **영속 상태 (Persistent State)**: 객체가 영속성 컨텍스트에 저장되어 있고, 데이터베이스와 연결된 상태를 말합니다. 이 상태의 객체는 변경 사항이 자동으로 데이터베이스에 반영됩니다.
- **임시 상태 (Transient State)**: 객체가 생성되었지만 영속성 컨텍스트에 저장되지 않은 상태입니다. 이 상태의 객체는 메모리에서만 존재합니다.
- **분리 상태 (Detached State)**: 영속성 컨텍스트에 의해 관리되던 객체가 컨테이너에서 분리된 상태입니다. 변경 사항은 자동으로 데이터베이스에 반영되지 않으며, 다시 영속성 컨텍스트에 연결해야 합니다.

## 생명 주기 관리

1. Create(생성): 객체가 생성되고, 영속성 컨텍스트에 저장됩니다.
2. Read(읽기): 객체가 영속성 컨텍스트에서 조회됩니다. 데이터베이스와의 동기화가 이루어집니다.
3. Update(수정): 객체의 속성이 변경되면 영속성 컨텍스트가 변경 사항을 감지하고, 트랜잭션이 커밋될 때 데이터베이스에 반영됩니다.
4. Delete(삭제): 객체가 영속성 컨텍스트에서 제거되면, 필요할 경우 데이터베이스에서 해당 레코드도 삭제됩니다.

## 플러시(Flush)

변경 사항을 데이터베이스에 반영하는 것. 명시적으로 호출하거나 트랜잭션이 커밋될 때 자동으로 실행됩니다.

## 트랜잭션 관리

여러 데이터 조작 작업을 하나의 단위로 묶어 관리합니다. 트랜잭션을 통해 데이터의 일관성을 유지합니다.

## 캐시 및 성능

메모리에 객체를 저장하여 재조회 시 성능을 향상시킵니다. Lazy Loading(지연 로딩)과 Eager Loading(즉시 로딩)을 선택하여 성능을 최적화할 수 있습니다.

## 비유를 통한 설명 - 도서관과 책

1. **도서관 (영속성 컨텍스트)**: 도서관 자체가 영속성 컨텍스트와 같습니다. 이곳에서 모든 책을 관리하고 그 상태를 추적합니다.
2. **책 (객체)**: 각 책은 데이터베이스에 저장되는 개별 객체를 나타냅니다. 각 책의 상태(대출 중, 서가에 있음, 분실 등)를 도서관에서 관리합니다.
3. **서가에 있는 책 (영속 상태)**: 언제든지 대출할 수 있고, 도서관과 연결된 상태입니다.
4. **대출된 책 (분리 상태)**: 대출자는 책을 가지고 있지만, 도서관의 관리 대상에서 벗어난 상태입니다. 반납하면 다시 도서관에 들어오게 됩니다.
5. **구매된 책 (임시 상태)**: 도서관에 들어오긴 했지만, 아직 서가나 대출 대상 목록에 포함되지 않은 새 책입니다.
6. **도서관의 시스템 업데이트 (플러시)**: 도서관은 모든 변경 사항을 새로운 상태로 기록할 때, 데이터베이스로 변화가 반영됩니다.
7. **시리즈 책 (연관 관계)**: 특정 작가의 모든 책이 서로 연결되어 있는 것처럼, 도서관은 연관 관계를 관리하여 특정 작가의 모든 책을 쉽게 찾을 수 있습니다.
8. **대출이 빈번한 인기 책 (캐시)**: 자주 대출되는 인기 책은 항상 서가에 잘 정리되어 있어 쉽게 찾을 수 있는 것처럼, 캐시는 재조회 시 성능을 높여줍니다.
