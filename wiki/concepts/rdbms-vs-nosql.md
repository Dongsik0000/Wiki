# RDBMS와 NoSQL

## RDBMS란?

RDBMS는 관계형 데이터베이스 관리 시스템으로, 데이터를 테이블 형식으로 저장하고, 다양한 테이블 간의 관계를 설정하여 데이터를 구조화하는 시스템입니다.

**특징:**
1. 테이블 기반: 데이터는 행과 열로 구성된 테이블에 저장됩니다.
2. 관계: 테이블 간의 관계를 설정할 수 있으며, 외래 키를 통해 이 관계를 정의합니다.
3. SQL 지원: 데이터 조회 및 조작을 위해 SQL을 사용합니다.
4. ACID 프로퍼티: 원자성(Atomicity), 일관성(Consistency), 고립성(Isolation), 지속성(Durability)을 지원합니다.
5. 데이터 무결성: 기본 키, 외래 키, 유니크 제약 등 데이터의 정확성과 일관성을 유지하기 위한 제약 조건이 있습니다.

## NoSQL (Not Only SQL)

NoSQL 데이터베이스는 비관계형 데이터베이스로, 비구조적 또는 반구조적 데이터를 저장하고 처리하는 방식입니다. 다양한 데이터 모델을 지원합니다.

**특징:**
1. 유연한 스키마: 스키마가 고정되어 있지 않아서, 데이터 구조를 개발 중에도 자유롭게 변경할 수 있습니다.
2. 수평 확장성: 데이터가 증가함에 따라 서버를 추가하여 시스템을 쉽게 확장할 수 있습니다.
3. 다양한 데이터 모델: 문서, 키-값, 열 패밀리, 그래프 데이터베이스 등 여러 가지 형식으로 데이터를 저장할 수 있습니다.
4. 고속 처리: 대량의 비정형 데이터를 빠르게 처리할 수 있는 성능을 제공합니다.
5. CAP 정리: Consistency(일관성), Availability(가용성), Partition tolerance(분할 내구성) 중에서 두 가지만 보장하는 CAP 정리를 유지합니다.
