# www.naver.com 접속 시 클라이언트-서버 흐름

**질문:** www.naver.com을 브라우저로 접속할 때, 발생하는 일을 클라이언트-서버 흐름으로 설명해주세요.

## 단계별 설명

1. **URL 입력**: www.naver.com을 입력
2. **Domain Name 확인 및 DNS 조회**: Domain Name을 추출 후 DNS 서버에 요청하여, 해당 도메인에 연결된 IP 주소를 조회하고 DNS 서버는 요청을 받아 해당 IP 주소를 반환
3. **HTTP 요청 메시지 생성**: 브라우저는 DNS 서버로부터 받은 IP 주소와 입력된 URL 정보를 사용해 HTTP 요청 메시지를 생성
4. **TCP 연결을 통한 HTTP 요청 전송**: 생성된 HTTP 요청 메시지는 TCP 프로토콜을 사용해 인터넷을 통해 전달
5. **서버에서 HTTP 요청 처리**: 서버는 도착한 HTTP 요청 메시지를 받아, HTTP 프로토콜을 사용해 메시지를 해석하고 해석된 요청 메시지에서 웹 페이지 URL 정보를 확인하고, 이에 해당하는 데이터를 검색
6. **HTTP 응답 메시지 생성**: 서버는 요청받은 데이터(웹 페이지)를 준비한 뒤, 이를 HTTP 응답 메시지로 변환. 변환된 HTTP 응답 메시지는 TCP 프로토콜을 사용해 인터넷을 통해 클라이언트(사용자의 컴퓨터)로 전송
7. **브라우저에서 응답 처리**: 클라이언트의 브라우저는 서버로부터 받은 HTTP 응답 메시지를 해석하고, 해석된 메시지에서 웹 페이지 데이터를 추출하여 브라우저 화면에 렌더링. 사용자는 요청한 웹 페이지를 볼 수 있게 됩니다.

## 추가 정보

- DNS란? 도메인 이름을 컴퓨터가 이해할 수 있는 IP 주소로 변환하는 시스템
- TCP란? 데이터를 인터넷을 통해 안전하고 신뢰성 있게 전송하기 위한 프로토콜(통신 규약)

## 짧은 요약

사용자가 브라우저에 주소를 입력하면, DNS 서버가 해당 주소에 맞는 IP를 찾아줍니다. 브라우저는 이 IP로 서버에 요청을 보내고, 서버는 요청한 데이터를 응답으로 보냅니다. 이 데이터는 브라우저가 처리해서 화면에 웹 페이지로 보여줍니다.
